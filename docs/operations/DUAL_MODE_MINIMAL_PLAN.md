# 在庫管理アプリ 二重運用（最小工程版）

## 目的
- 同じアプリを Web と デスクトップ の2通りで運用する。
- 改修は最小限（ほぼゼロ）にする。

## 方針（最小工程）
- バックエンドは今の Go サーバーをそのまま使う。
- 画面は今の Web 画面をそのまま使う。
- デスクトップ版は「専用ブラウザ窓（アプリモード）」で運用する。

この方針なら、Electron や Wails への移植が不要で、最短で二重運用ができる。

## 実施タスク（順番どおり）

1. サーバー起動コマンドを固定する
- 実行: go run main.go
- 期待結果: http://localhost:8080 で画面が開ける

2. Web運用手順を確定する
- 社内PCのブラウザで http://サーバーIP:8080 にアクセス
- ルーターやWindows Defenderで 8080 ポートを許可
- 運用手順書に「URL・起動手順・停止手順」を記載

3. デスクトップ運用を追加する（改修なし）
- 端末ごとにブラウザのアプリモード起動ショートカットを作成
- 例（Edge）:
  - msedge.exe --app=http://localhost:8080
- 例（Chrome）:
  - chrome.exe --app=http://localhost:8080
- これで通常タブではなく、アプリ風の専用ウィンドウで運用できる

4. 自動起動を設定する（任意だが推奨）
- Windows タスクスケジューラでログオン時に以下を起動
  - Go サーバー
  - Edge/Chrome のアプリモード
- 端末起動後に自動で業務画面が立ち上がるようにする

5. DB運用ルールを最低限決める
- 現状の参照DBを統一する（今は inventory_test.db を参照）
- バックアップを日次でコピー
- 退避先フォルダを固定（例: D:\backup\souko_2）

6. 動作確認（10分チェック）
- Web: 一般ブラウザで開ける
- Desktop: アプリモード窓で開ける
- 共通: 入庫・出庫・廃棄・バーコード検索が通る
- 共通: 同じデータが見える（DBが同一である）

## 改修が必要な場合（最小）
- 基本は改修不要。
- 必要なら次の1点だけ実施:
  - DBパスを環境変数で切替可能に戻す（開発/本番の誤接続防止）

## ここまでの成果物（最小）
- Web運用: URLで利用可能
- デスクトップ運用: アプリモードショートカットで利用可能
- コードの大規模変更なし
